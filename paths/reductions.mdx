Reductions are one of the foundational patterns in parallel computing. Given a set of values and an associative binary operator, a reduction collapses the set into a single output. Sum, product, min, max, bitwise AND/OR — all reductions of one input vector.

On a GPU, the challenge is *not* the arithmetic. A reduction does $N$ operations to consume $N$ values; arithmetic-wise it's trivial. The real game is parallelism and memory: a naïve serial loop has $O(N)$ latency, while a tree reduction has $O(\log N)$ latency given enough threads. The art is mapping that tree onto threads, blocks, and warps so that bandwidth and the warp-level primitives are saturated rather than the arithmetic units.

By the end you'll know when to roll your own and when to reach for a library primitive.
